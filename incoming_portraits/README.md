# Папка-приёмник готовых портретов

Сюда кладёшь портреты операторов, подготовленные в Photoshop по шаблону
(600×600, квадрат, белый фон — см. `tools/README.md`, раздел «Шаблон
портрета в Photoshop»). Дальше просто пишешь в сессии с Claude Code
что-то вроде «портреты для <команда> готовы, лежат в
incoming_portraits» — остальное (вырезание фона, раскладка по `docs/`,
привязка к `game_data.json`, синхронизация) я делаю сам.

## Структура

Один подкаталог на команду, имя подкаталога = точное название команды
как в игре (как оно должно быть в `game_data.json`, например
`Nemesis Claw`). Внутри — файлы с именами операторов:

```
incoming_portraits/
└─ Nemesis Claw/
   ├─ Night Lord Visionary.jpg
   ├─ Night Lord Fearmonger.jpg
   └─ ...
```

Имя файла (без расширения) = точное имя оператора, под которым он
заведён (или будет заведён) в `game_data.json`. Регистр и пробелы как в
игре — по этому имени идёт сопоставление, лишние подписи не нужны.

Формат файла — JPG или PNG, без разницы, всё равно пересохраняется в
PNG с прозрачным фоном.

## Что происходит дальше

Обработанные файлы попадают в `docs/img/portraits/<team-slug>/` (уже
без белого фона), эта папка (`incoming_portraits/`) после обработки
очищается — она рабочая, не архив. В git не коммитится
(`.gitignore` в корне проекта).
