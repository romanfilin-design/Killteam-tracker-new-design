# Папка-приёмник готовых портретов

Сюда кладёшь фото миниатюр операторов — любой формат (jpg/jpeg/png/webp),
любой фон (студийный, трава на подставке, тени — фон снимается
автоматически через rembg, не обязательно чистый белый), фигурка видна
целиком и не перекрыта другими моделями/текстом. Если на фото несколько
миниатюр — кадрируй сам до одной перед тем как класть сюда. Дальше
просто пишешь в сессии с Claude Code что-то вроде «портреты для
<команда> готовы, лежат в incoming_portraits» — остальное (вырезание
фона, раскладка по `docs/`, привязка к `game_data.json`, синхронизация)
я делаю сам (см. `tools/README.md`, раздел `smart_crop.py`).

## Структура

Один подкаталог на команду, имя подкаталога = точное название команды
как в игре (как оно должно быть в `game_data.json`, например
`Nemesis Claw`). Внутри — файлы с именами операторов:

```
incoming_portraits/
└─ Nemesis Claw/
   ├─ Night Lord Visionary.jpg
   ├─ Night Lord Fearmonger.jpg
   └─ ...
```

Имя файла (без расширения) = точное имя оператора, под которым он
заведён (или будет заведён) в `game_data.json`. Регистр и пробелы как в
игре — по этому имени идёт сопоставление, лишние подписи не нужны.

## Что происходит дальше

Обработанные файлы попадают в `docs/img/portraits/<team-slug>/` (уже
без фона), эта папка (`incoming_portraits/`) после обработки
очищается — она рабочая, не архив. В git не коммитится
(`.gitignore` в корне проекта).
