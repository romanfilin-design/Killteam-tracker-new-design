# Добавление новой команды — инструкция

Что автоматизировано скриптами в этой папке, а что всё равно требует
разбора руками (в сессии с Claude Code).

## Что нельзя (пока) полностью автоматизировать

Перевод правил из PDF и превращение их в структуру `game_data.json`
требует понимания контекста (что из текста правил идёт в характеристики
оператора, что — в описание снаряжения, как сформулировать краткий
русский текст для карточки-памятки). Это шаг, который делает Claude в
сессии, читая PDF и картинки.

## Важно: кэш браузера после обновления CSS/JS

`docs/index.html` подключает `css/style.css`, `js/app.js` и остальные
скрипты с query-параметром `?v=...` — это принудительно заставляет
браузер (и GitHub Pages/CDN) подтянуть новый файл вместо закэшированной
старой версии. **При любом изменении CSS/JS нужно вручную поднять это
значение** во всех трёх `<link>`/`<script>` тегах в `docs/index.html`
(например, на текущую дату) — иначе пользователи с уже открытым сайтом
могут долго видеть старую версию интерфейса, хотя `game_data.json` (без
версии в URL, подгружается через `fetch`) обновится сразу.

## Что автоматизировано

### 1. `sync_release.py` — синхронизация `docs/`

После того как `game_data.json` в корне проекта отредактирован (Claude
это делает сам), нужно скопировать актуальные данные в `docs/` и
пересобрать встроенный фолбэк. Раньше это делалось руками через
python-однострочники в чате — теперь один скрипт:

```bash
python3 tools/sync_release.py
```

Проверяет структуру (обязательные ключи, ссылки архетипов на
`tacOpsByArchetype`, ссылки `equipment` на `factionEquipmentByTeam`) и
падает с понятной ошибкой, если что-то не так — не даст залить
битый JSON.

### 2. `smart_crop.py` — кроп портретов

Официальные фото миниатюр (например, с сайта Games Workshop) обычно на
белом фоне с большим полем вокруг фигурки. Скрипт находит саму
фигурку, отрезает белое поле и кадрирует квадрат по голове/плечам, а не
по всей фигуре целиком.

Одна картинка:

```bash
python3 tools/smart_crop.py "input/Night Lord Warrior.jpg" docs/img/portraits/nemesis-claw/warrior.jpg
```

Вся папка сразу (имя файла превращается в slug — пробелы на `_`,
нижний регистр):

```bash
python3 tools/smart_crop.py --batch "input_dir/" docs/img/portraits/<team-slug>/
```

Если авто-кроп утянул кадр в сторону (так было с Ventrilokar — рядом со
шлемом стоял штандарт, и алгоритм центрировался по нему, а не по
голове) — открой исходник, прикинь координаты на глаз и задай кроп
руками:

```bash
python3 tools/smart_crop.py "input/Night Lord Ventrilokar.jpg" \
  docs/img/portraits/nemesis-claw/ventrilokar.jpg \
  --manual-box 350,260,660,570
```

`--manual-box LEFT,TOP,RIGHT,BOTTOM` — пиксельные координаты
прямоугольника в исходном (некадрированном) изображении.

## Полный пошаговый пайплайн добавления новой команды

1. **Собери материалы**: PDF с правилами команды на английском (карты
   операторов, состав отряда, снаряжение фракции) + фотографии
   миниатюр, по одной на оператора, с понятными именами файлов (имя
   оператора как в PDF — так Claude однозначно сопоставит файл с
   картой).
2. **Открой сессию Claude Code** в этой папке проекта, приложи PDF и
   фото (можно частями — Claude подождёт, если явно сказать, что
   пришлёшь ещё материалы).
3. Скажи «старт» (или любую явную команду начать обработку). Claude:
   - прочитает PDF (характеристики операторов, состав отряда, архетипы,
     снаряжение фракции, спец-правила);
   - сопоставит фото с операторами по именам файлов;
   - переведёт нужные текстовые поля на русский и допишет команду в
     `game_data.json` в корне проекта (по той же структуре, что уже
     используют существующие команды);
   - скопирует фото портретов в `docs/img/portraits/<team-slug>/` и
     прогонит их через `smart_crop.py` (с ручной правкой, если авто-кроп
     не сработал для конкретной позы);
   - запустит `python3 tools/sync_release.py`, чтобы обновить
     `docs/game_data.json` и фолбэк;
   - проверит результат в браузере (preview): выбор команды, лимиты
     пула, старт партии, порядок отображения портретов.
4. **Ты просматриваешь `docs/`** (при желании — открываешь
   `docs/index.html` через локальный сервер) и, если всё ок, делаешь
   `git add` / `git commit` / `git push` сам.

Шаги 3 (кроме собственно перевода/анализа PDF) можно попросить Claude
выполнить и без пересказа этого списка — достаточно приложить PDF и
фото и написать «добавь команду».
