# Добавление новой команды — инструкция

Что автоматизировано скриптами в этой папке, а что всё равно требует
разбора руками (в сессии с Claude Code).

## Что нельзя (пока) полностью автоматизировать

Перевод правил из PDF и превращение их в структуру `game_data.json`
требует понимания контекста (что из текста правил идёт в характеристики
оператора, что — в описание снаряжения, как сформулировать краткий
русский текст для карточки-памятки). Это шаг, который делает Claude в
сессии, читая PDF и картинки.

## Важно: кэш браузера после обновления CSS/JS

`docs/index.html` подключает `css/style.css`, `js/app.js` и остальные
скрипты с query-параметром `?v=...` — это принудительно заставляет
браузер (и GitHub Pages/CDN) подтянуть новый файл вместо закэшированной
старой версии. **При любом изменении CSS/JS нужно вручную поднять это
значение** во всех трёх `<link>`/`<script>` тегах в `docs/index.html`
(например, на текущую дату) — иначе пользователи с уже открытым сайтом
могут долго видеть старую версию интерфейса, хотя `game_data.json` (без
версии в URL, подгружается через `fetch`) обновится сразу.

## Что автоматизировано

### 1. `sync_release.py` — синхронизация `docs/`

После того как `game_data.json` в корне проекта отредактирован (Claude
это делает сам), нужно скопировать актуальные данные в `docs/` и
пересобрать встроенный фолбэк. Раньше это делалось руками через
python-однострочники в чате — теперь один скрипт:

```bash
python3 tools/sync_release.py
```

Проверяет структуру (обязательные ключи, ссылки архетипов на
`tacOpsByArchetype`, ссылки `equipment` на `factionEquipmentByTeam`) и
падает с понятной ошибкой, если что-то не так — не даст залить
битый JSON.

### 2. `smart_crop.py` — подготовка портретов (кроп + вырезание фона)

Всегда сохраняет результат как PNG с прозрачным фоном (белый фон
студийных фото иначе спорит с тёмной темой на крупных карточках —
см. `docs/img/portraits/nemesis-claw/` для примера).

**Основной сценарий теперь** — портреты готовятся заранее в Photoshop
уже квадратом нужного размера (см. «Шаблон портрета в Photoshop» ниже),
скрипту остаётся только вырезать фон:

```bash
python3 tools/smart_crop.py "incoming_portraits/Nemesis Claw/Night Lord Warrior.jpg" \
  docs/img/portraits/nemesis-claw/warrior.png --no-crop
```

Пакетно на всю папку команды:

```bash
python3 tools/smart_crop.py --batch "incoming_portraits/Nemesis Claw/" \
  docs/img/portraits/nemesis-claw/ --no-crop
```

**Запасной сценарий** — если прислали необработанное фото (белый фон,
фигурка не по центру, как бывает у официальных фото GW) — без
`--no-crop` скрипт сам найдёт баунд-бокс фигурки и кадрирует квадрат по
голове/плечам:

```bash
python3 tools/smart_crop.py input.jpg docs/img/portraits/<team-slug>/name.png
```

Если авто-кроп утянул кадр в сторону (так было с Ventrilokar — рядом со
шлемом стоял штандарт, и алгоритм центрировался по нему, а не по
голове) — открой исходник, прикинь координаты на глаз и задай кроп
руками:

```bash
python3 tools/smart_crop.py "input/Night Lord Ventrilokar.jpg" \
  docs/img/portraits/nemesis-claw/ventrilokar.png \
  --manual-box 350,260,660,570
```

`--manual-box LEFT,TOP,RIGHT,BOTTOM` — пиксельные координаты
прямоугольника в исходном (некадрированном) изображении. Флаг
`--no-alpha` отключает вырезание фона, если для какой-то команды оно
не нужно.

### Шаблон портрета в Photoshop

- **Холст: 600×600 px, квадрат, RGB.**
  На экране подготовки портрет показывается на ширину ~156px (CSS), на
  игровом экране — 44px; 600px даёт запас под retina-экраны и на случай,
  если портрет где-то в будущем станет крупнее, но не настолько много,
  чтобы раздувать вес страницы при загрузке карусели из 6–8 карточек.
- Миниатюра занимает большую часть кадра, отступы по краям равномерные
  (~5–8% от стороны) — не нужно вплотную резать по контуру, вырезание
  фона всё равно уберёт лишнее белое поле.
- Фон — сплошной белый `#FFFFFF`, без градиента и виньетки (иначе
  вырезание фона сработает хуже — алгоритм ищет именно чистый белый).
- Экспорт — JPG (качество ~90) или PNG, без разницы: скрипт всё равно
  пересохраняет в PNG с альфа-каналом.
- **Имя файла = точное имя оператора**, как оно заведено (или будет
  заведено) в `game_data.json`, например `Night Lord Warrior.jpg` — по
  нему идёт сопоставление файла с данными.

### Пайплайн через папку-приёмник `incoming_portraits/`

Готовые портреты кладутся в `incoming_portraits/<Название команды>/`
(см. `incoming_portraits/README.md` — там та же инструкция для быстрой
сверки). Дальше в сессии с Claude Code достаточно написать что-то вроде
«портреты для Nemesis Claw готовы, лежат в incoming_portraits» — Claude:

1. проверит файлы (квадрат, разумное разрешение, имена совпадают с
   операторами команды в `game_data.json`);
2. прогонит их через `smart_crop.py --no-crop` (или без флага, если
   видно, что кроп не готов) в `docs/img/portraits/<team-slug>/`;
3. если у команды в `game_data.json` ещё нет ссылок `portrait` на эти
   файлы — допишет их;
4. запустит `python3 tools/sync_release.py`;
5. проверит результат в браузере;
6. очистит `incoming_portraits/<Название команды>/` (она рабочая, не
   архив, в git не коммитится).

## Полный пошаговый пайплайн добавления новой команды

1. **Собери материалы**: PDF с правилами команды на английском (карты
   операторов, состав отряда, снаряжение фракции) + портреты миниатюр,
   подготовленные по шаблону (600×600, квадрат, белый фон — см. выше),
   с именами файлов = имена операторов как в PDF.
2. Портреты сразу складываешь в `incoming_portraits/<Название
   команды>/` (см. отдельный раздел выше). PDF — прикладываешь прямо в
   сессию Claude Code в этой папке проекта.
3. Скажи «старт» (или любую явную команду начать обработку). Claude:
   - прочитает PDF (характеристики операторов, состав отряда, архетипы,
     снаряжение фракции, спец-правила);
   - сопоставит файлы из `incoming_portraits/` с операторами по именам;
   - переведёт нужные текстовые поля на русский и допишет команду в
     `game_data.json` в корне проекта (по той же структуре, что уже
     используют существующие команды);
   - прогонит портреты через `smart_crop.py --no-crop` в
     `docs/img/portraits/<team-slug>/`;
   - запустит `python3 tools/sync_release.py`, чтобы обновить
     `docs/game_data.json` и фолбэк;
   - проверит результат в браузере (preview): выбор команды, лимиты
     пула, старт партии, порядок отображения портретов.
4. **Ты просматриваешь `docs/`** (при желании — открываешь
   `docs/index.html` через локальный сервер) и, если всё ок, делаешь
   `git add` / `git commit` / `git push` сам.

Шаги 3 (кроме собственно перевода/анализа PDF) можно попросить Claude
выполнить и без пересказа этого списка — достаточно приложить PDF и
фото и написать «добавь команду».
