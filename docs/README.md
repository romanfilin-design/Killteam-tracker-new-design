# Kill Team Solo Tracker

Мобильный веб-трекер одной команды для настольной игры Kill Team. Ведёт
подготовку состава и партию в реальном времени: здоровье, приказы,
статусы, снаряжение, счётчики. Чистый HTML/CSS/JS, без сборки —
разворачивается как статический сайт (GitHub Pages).

## Структура

- `index.html` — точка входа.
- `css/style.css` — визуальный стиль (grimdark sci-fi терминал оператора).
- `js/tracker_logic.js` — бизнес-логика (создание сущностей, переходы
  состояния, сериализация). Ни строчки разметки/CSS.
- `js/app.js` — слой рендера и обработки событий, связывает логику с DOM.
- `game_data.json` — игровые данные (Crit Op, Kill Op, Tac Op, снаряжение,
  Kill Teams). Подгружается через `fetch('./game_data.json')`.
- `js/game_data_fallback.js` — встроенная копия тех же данных на случай,
  если `fetch` недоступен (например, открытие файла напрямую с диска без
  локального сервера).

## Запуск локально

Открыть `index.html` через любой статический сервер, например:

```
python3 -m http.server 8000
```

и перейти на `http://localhost:8000/`.

## Деплой на GitHub Pages

1. Запушить содержимое этой папки в репозиторий (например, в ветку `main`
   или в отдельную ветку `gh-pages`, либо в поддиректорию `docs/`).
2. В настройках репозитория GitHub → Pages выбрать источник (ветка/папка,
   где лежит `index.html`).
3. Открыть выданный GitHub Pages URL.

## Данные

Состояние партии хранится в `localStorage` браузера устройства и
переживает перезагрузку страницы. Есть экспорт/импорт всего состояния в
JSON-файл (кнопки «Экспорт»/«Импорт» на экране игры) и отдельный импорт
ростера операторов на экране подготовки.
