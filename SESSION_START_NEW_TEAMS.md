# Стартовый бриф — добавление новых Kill Team

Вставь этот файл целиком первым сообщением в новую сессию Claude Code,
открытую в папке `Killteam new design`. Он даёт всё нужное, чтобы сразу
добавлять команды, не восстанавливая контекст с нуля.

## Что это за проект

Kill Team Solo Tracker — статический сайт (без бэкенда, без build-шага)
для соло-игры в настолку Warhammer 40,000: Kill Team. Прод:
https://romanfilin-design.github.io/Killteam-tracker-new-design/

Стек: ванильный JS (IIFE, без фреймворка), данные в `localStorage`,
хостинг — GitHub Pages из папки `docs/` через собственный workflow
`.github/workflows/pages.yml` (GitHub Actions, недавно настроено — до
этого была легаси-сборка Jekyll «Deploy from a branch», которая
периодически зависала; теперь деплой быстрый и надёжный).

## Главный документ — прочитай его первым

**`tools/README.md`** — полная пошаговая инструкция добавления команды:
что автоматизировано (`sync_release.py`, `smart_crop.py`), конвенция
перевода правил, обязательный шаг вычитки перед мёржем, структура
`factionChoices`/`statusTokens` в данных. Всё ниже — сжатая выжимка
самых легко забываемых моментов, но READMЕ — источник истины.

## Пайплайн одним взглядом

1. PDF с правилами команды кладётся в `incoming_teams/<Название>.pdf`.
2. Портреты (если готовы) — в `incoming_portraits/<Название>/`, имя
   файла = точное имя оператора.
3. По команде «добавь команду <Название>» — разбор PDF (много картинок
   датакард, тяжело по контексту) делегируется отдельному субагенту.
   **Не доверяй его черновику вслепую** — в этой сессии субагенты не
   раз ошибались: забывали `maxCopies` у повторяемых операторов,
   оставляли `&amp;` вместо `&` в архетипах (ломает сопоставление с
   `tacOpsByArchetype`), вставляли мета-комментарии «для трекера» прямо
   в текст правил, который увидит игрок. Делай отдельный проверочный
   проход перед записью в `game_data.json`.
4. **Обязательное правило перевода**: связки/глаголы/обычные
   существительные — на русском; официальные игровые термины и
   ключевые слова (Range, Piercing, Lethal, APL, Move, Save, Wounds,
   Atk, Dmg, Hit, control range, killzone, turning point, Strategic
   Gambit, Charge, Dash, FLY и т.п.) — остаются английскими внутри
   русского текста. Названия оружия — тоже английские.
5. **Обязательное правило про лор**: в оригинале почти каждое описание
   способности/faction rule/снаряжения начинается с одного предложения
   или абзаца курсивом — художественное вступление без механики. Его
   убираем полностью, оставляем только механический текст. Иногда это
   всего одно предложение — вычитывай каждую запись, не только длинные.
   Секция `lore` (`intro`/`operatives[].text`) — исключение, она и
   задумана как художественный текст.
6. Если у команды есть механика «выбери N из списка» на уровне команды
   (не оператора) — оформляй через `factionChoices` в `game_data.json`
   (см. Angels of Death как референс): `scope: 'setup'` — выбор один раз
   при сборке (Chapter Tactics), `scope: 'game'` — можно менять по ходу
   партии (Combat Doctrine). У UI уже есть обе панели
   (`renderFactionChoicesSetupPanel`/`renderFactionChoicesGamePanel` в
   `docs/js/app.js`) — просто описать данные, код трогать не надо.
7. Если у команды есть особый токен/статус — заводи `statusTokens`
   (`{id, name, target: 'friendly'|'enemy'}`). `target:'friendly'`
   попадает в быстрые чипы на карточке оператора. `target:'enemy'`
   (Poison у Plague Marines, Grudge у Hearthkyn) — данные сохраняем, но
   чипы **не рендерим**: трекер сейчас ведёт только свою команду, это
   задел под будущую комнату синхронизации игроков (см.
   `SESSION_START_DEVELOPMENT.md`, там это следующий большой пункт).
8. Проверь: не появились ли в датакартах weapon rules, которых нет в
   `TERM_GLOSSARY` (`docs/js/app.js`, ищи по имени константы) — если
   появились, добавь короткое объяснение в том же тоне, что соседние
   записи.
9. Портреты: `python3 tools/smart_crop.py --batch "incoming_portraits/<Команда>/" docs/img/portraits/<team-slug>/`
   — используется `rembg` (ML, любой фон) из общего окружения
   `Warhammer/.venv-tools/`, общего с проектом Spearhead. Не требует
   специальной подготовки фото, кроме «миниатюра видна целиком».
10. `python3 tools/sync_release.py` — копирует `game_data.json` и
    `tracker_logic.js` из корня в `docs/`, пересобирает встроенный
    фолбэк, и **валидирует структуру** (упадёт с понятной ошибкой на
    битых ссылках архетипов/снаряжения).
11. **Подними версию кэша** в `docs/index.html` — три `?v=...` в
    `<link>`/`<script>` тегах — на любую новую дату/суффикс. Без этого
    браузер может показывать старую версию JS/CSS сколько угодно долго.
12. Проверь в браузере (см. ниже) — выбор команды, сборка ростера,
    лимиты копий, старт партии, датакарта, faction-choices если есть.
13. Очисти `incoming_teams/<Название>.pdf` и `incoming_portraits/<Название>/`
    **после явного подтверждения пользователя** — это рабочие папки, не
    архив, но удаление файлов — необратимое действие, спрашивай прежде
    чем удалять.

## Проверка в браузере

Есть Claude Preview tools (`preview_start` с конфигом `docs-static` из
`.claude/launch.json`, порт 8420). `preview_console_logs` в этой сессии
иногда залипал на старых записях после нескольких reload — не
единственный источник правды; перепроверяй через `preview_screenshot`
и прямой `localStorage.getItem('kt_solo_tracker_state_v1')` через
`preview_eval`, если подозреваешь баг.

## Git / деплой

- Remote уже на SSH (`git@github.com:romanfilin-design/Killteam-tracker-new-design.git`),
  push работает без пароля.
- **Никогда не делай `git commit`/`git push` без явного «деплой»/«залей»/
  «заливай» от пользователя в этом же сообщении** — это твёрдое правило
  этого проекта, даже если до этого пуш уже разрешали.
- `gh` CLI установлен и авторизован на этой машине (fine-grained PAT в
  keychain) — можно смотреть `gh run list`, `gh api repos/.../pages`
  и т.п., если деплой выглядит странно. Токен может быть отозван
  пользователем в любой момент — если авторизация пропала, это нормально,
  просто скажи пользователю.
