# Папка-приёмник PDF новых команд

Сюда кладёшь официальный PDF с правилами команды (карты операторов,
состав отряда, архетипы, снаряжение фракции). Дальше пишешь в сессии с
Claude Code что-то вроде «добавь команду <Название>, PDF лежит в
incoming_teams» — я сам читаю файл по пути и разбираю его; прикладывать
PDF в чат не нужно.

## Структура

Один файл на команду, имя файла = точное название команды, как оно
должно быть в игре (совпадает с ключом в `game_data.json` и с именем
папки в `incoming_portraits/`):

```
incoming_teams/
├─ Nemesis Claw.pdf
└─ Blooded.pdf
```

Формат не важен (PDF с текстом или со сканами/картинками) — я читаю
постранично.

## Портреты — отдельно

Если вместе с PDF готовы и портреты операторов, они всё равно кладутся
в свою обычную папку `incoming_portraits/<Название команды>/` (см.
`incoming_portraits/README.md`), не сюда. Просто упомяни в сообщении,
что портреты тоже готовы — я подхвачу обе папки за один проход.

## Что происходит дальше

Я разбираю PDF (в отдельном субагенте, чтобы не раздувать контекст
текущей сессии постраничными картинками), перевожу нужные текстовые
поля на русский, дописываю команду в `game_data.json`, обрабатываю
портреты (если они есть), прогоняю `tools/sync_release.py` и проверяю
результат в браузере. После обработки эта папка очищается — она
рабочая, не архив, в git не коммитится.
